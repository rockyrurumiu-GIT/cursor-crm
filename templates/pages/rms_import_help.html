批量导入候选人

1. 一键导入 dry-run（预览） 不写库 · 不存文件

先预览，不写数据库、不保存简历文件。

./venv/bin/python scripts/import_rms_resumes_with_ocr_retry.py \
  --source-dir "/你的简历目录" \
  --recursive \
  --dry-run
2. 一键导入 commit（正式入库） 写入候选人与简历

确认 dry-run 报告无误后再执行。仅写入候选人与简历，不会自动推荐到岗位。可选：--limit 限制导入文件数；--ocr-limit 限制 OCR 尝试数。

./venv/bin/python scripts/import_rms_resumes_with_ocr_retry.py \
  --source-dir "/你的简历目录" \
  --recursive \
  --commit
3. 导入与 OCR 结果含义 status 字段

一键流程会分别输出文本导入与 OCR retry 的统计。报告行 status 常见值：

  • created：文本解析或 OCR 后姓名+电话齐全，成功新建候选人及简历（dry-run 时为 would_create）
  • updated：OCR 为已有候选人补全缺失的姓名/电话（dry-run 时为 would_update）
  • skipped_duplicate：姓名 + 手机号与库内或本批次重复，跳过
  • skipped_unparseable：文本解析或 OCR 后仍缺姓名/电话，或姓名校验不通过
  • failed：读文件失败、文件过大、Tesseract 异常、数据库写入错误等
4. 查看导入报告 reports/rms_imports/

报告默认输出到 reports/rms_imports/，包含 CSV 与 JSON 摘要。报告中的手机号、邮箱已脱敏。

  • 文本导入：rms_resume_import_YYYYMMDD_HHMMSS.csv/json
  • OCR retry：rms_resume_ocr_retry_YYYYMMDD_HHMMSS.csv/json
  • 一键合并摘要：rms_resume_import_with_ocr_retry_YYYYMMDD_HHMMSS.json
  • 修正 CSV：rms_resume_correction_YYYYMMDD_HHMMSS.csv/json
5. corrections.csv（OCR 后仍错误的记录） 人工修正

仅用于一键导入 + OCR 后仍错误的行（如 skipped_unparseable、conflict 等需人工修正的记录）。

resume_file_path,action,candidate_id,name,phone,email_wechat,school,major,education_level,source
/你的简历目录/a.pdf,create,,张三,13800138000,zhangsan@example.com,西安电子科技大学,计算机,本科,批量导入
,update,123,李四,13900139000,lisi@example.com,许昌学院,软件工程,本科,批量导入
/你的简历目录/c.pdf,skip,,,,,,,

action：create（新建候选人+简历）、update（按 candidate_id 更新非空字段）、skip（跳过）。

6. 修正 CSV dry-run 预览 
./venv/bin/python scripts/apply_rms_resume_import_corrections.py \
  --csv "/你的修正CSV路径/rms_resume_corrections.csv" \
  --dry-run
7. 修正 CSV commit 正式入库 
./venv/bin/python scripts/apply_rms_resume_import_corrections.py \
  --csv "/你的修正CSV路径/rms_resume_corrections.csv" \
  --commit
8. 调试 / 排障（单独脚本） 非日常入口

以下脚本不是日常批量导入入口，仅在需要拆分排查时使用：

  • scripts/import_rms_resumes.py — 仅文本解析，不自动 OCR
  • scripts/retry_rms_resume_ocr.py — 对已有导入报告 CSV 单独跑 OCR retry

示例（仅文本导入预览）：

./venv/bin/python scripts/import_rms_resumes.py \
  --source-dir "/你的简历目录" \
  --recursive \
  --dry-run

示例（单独 OCR retry 预览）：

./venv/bin/python scripts/retry_rms_resume_ocr.py \
  --report-csv "reports/rms_imports/rms_resume_import_YYYYMMDD_HHMMSS.csv" \
  --dry-run
注意事项 务必先 dry-run
  • 务必先 dry-run，再 commit。
  • 不要把真实简历、导入报告、修正 CSV 提交到 GitHub；reports/rms_imports/ 已在 .gitignore 中忽略。
  • 导入只写 rms_candidates 与 rms_resumes，不会自动推荐到岗位。
  • 去重规则：仅当姓名 + 手机号同时相同才判定重复。
